# 企业端面试模块优化记录

> 记录日期：2026-05-08
> 基线文档：企业端面试访谈模块审查报告.md

## 1. 本轮已完成项

### 1.1 路由级异常恢复

- 已在 /chat 路由外侧接入 ErrorBoundary。
- 新增异常恢复 fallback，可执行“重新加载并尝试恢复”。
- 目标是让渲染异常不再直接白屏，并衔接当前已存在的会话恢复能力。

### 1.2 数字人真正挂载到企业端面试页

- 之前 ChatPage 仅保留了数字人状态变量和面板外观，未真正渲染 VirtualHumanSDK。
- 现已将 VirtualHumanSDK 挂到企业端页面，数字人可以参与主链路。

### 1.3 数字人改为跟随主面试链路播报

- useInterview 仍是企业端面试唯一主状态机。
- 新增纯播报能力 speakText，避免继续依赖旧的 VirtualHumanSDK 内部面试流程。
- 当前问题和结束语可通过数字人播报。

### 1.4 旧状态机局部清理

- 修复 useInterviewState 中 externalInterviewQuestions 的闭包过期风险。
- 删除空的死代码 useEffect。

### 1.5 新增与跑通测试

- 新增 /chat 异常恢复测试。
- 新增数字人面板挂载测试。
- 测试结果：2 个 suite、3 个用例全部通过。
- 同时清理了测试中的 act/WebRTC 噪声输出。

### 1.6 数字人体验增强

- 面板新增连接中、等待交互、连接失败、未连接等状态提示。
- 新增手动重试连接按钮。
- 新增播报开关与静音控制。

### 1.7 数字人桥接层雏形

- 新增 useVirtualHumanBridge hook。
- 从 ChatPage 中抽离了数字人状态、播报、副作用、重试逻辑和部分诊断事件。
- 面板新增最近诊断摘要展示。

### 1.8 设计基线补齐

- 新增 企业端虚拟人统一状态机设计稿.md。
- 明确后续要走“单一主状态机 + 数字人桥接层 + SDK 能力化”的迁移方向。

### 1.9 企业端面试会话增量同步第一版

- useInterview 已接入 /api/interview-sessions 的 POST / PATCH。
- 当前会在 start_interview、question_generated、answer_submitted、next_question_generated、interview_completed 这 5 个 checkpoint 做后端同步。
- 同步 payload 已补齐 ownerUserId，并把 totalQuestions、currentQuestionIndex、lastSyncReason 等元数据纳入 checkpoint。

### 1.10 useSDKConnection 去旧状态依赖第一步

- frame_start / frame_stop 已不再读写旧 interviewState。
- SDK 连接层残余已从“直接驱动旧状态机”收敛为“仅处理 SDK 事件与桥接回调”。

### 1.11 InterviewPage 废弃清理落地

- 废弃的 InterviewPage.js 已从前端代码中删除。
- 当前企业端面试页统一收口到 /chat 主链路。

### 1.12 会话同步单测补齐

- 新增 useInterview.sync.test.jsx。
- 已覆盖“创建会话并同步首题 checkpoint”“答题后同步 answer_submitted / next_question_generated”“完成面试时同步 completed checkpoint”。
- 后续又补齐“token 缺失跳过同步但流程继续”“PATCH 失败不阻断答题推进”两条异常分支。
- 测试结果：1 个 suite、5 个用例全部通过。

### 1.13 VirtualHumanSDK 内部旧状态机完成删除

- VirtualHumanSDK 已不再引用 useInterviewState。
- useInterviewState.js 已从代码库删除。
- 数字人 SDK 组件对外只保留连接、播报、录音等能力型接口。

### 1.14 语音回答闭环第一版

- 复用了 ChatPage 现有浏览器语音识别能力。
- 语音转写结果会先进入“语音回答已转写”确认条。
- 当前支持“提交回答”“继续编辑”“稍后”三个动作，并可接入现有 sendMessage / submitAnswer 主链路。

### 1.15 语音确认条交互测试补齐

- 新增 InputBar.voice-answer.test.jsx。
- 已覆盖确认条展示和三个动作按钮回调。
- 测试结果：1 个 suite、2 个用例全部通过。

### 1.16 InterviewScoring 默认调试日志降噪

- 已关闭 InterviewScoring 默认 debug 输出。
- 仍保留 window / localStorage 显式开关，便于需要时手动打开调试。

### 1.17 SDK 原生 ASR 已接入数字人桥接层

- useSDKConnection 已支持从 SDK ASR 事件中提取 transcript，并通过 onAsrMessage 往上层回调。
- useVirtualHumanBridge 已接收 SDK 原生 ASR 文本，并以 sdk-asr 来源转发到现有 voice draft 通道。
- ChatPage 当前已让 browser-asr 与 sdk-asr 共用同一份语音回答确认闭环。

### 1.18 SDK ASR bridge 窄单测补齐

- 新增 useVirtualHumanBridge.asr.test.jsx。
- 已覆盖“收到 SDK 原生 transcript 后，以 sdk-asr 来源上抛，并记录 asr-transcript 诊断事件”。
- 测试结果：1 个 suite、1 个用例通过。

### 1.19 语音回答草稿状态继续下沉到 bridge

- voiceAnswerDraft、ASR 写入、继续编辑、稍后关闭已从 ChatPage 下沉到 useVirtualHumanBridge。
- ChatPage 当前只保留 sendMessage / submitAnswer 相关的提交主链路，不再直接维护语音草稿状态。
- browser-asr 与 sdk-asr 现在都通过 bridge 统一写入同一份语音回答草稿。

### 1.20 企业端面试会话周期心跳第一版

- useInterview 已在面试活跃期增加轻量心跳 PATCH。
- 心跳同步只更新 metadata，不携带 conversation，当前会补齐 lastHeartbeatAt、currentQuestionIndex、totalQuestions。
- useInterview.sync.test.jsx 已新增心跳测试，验证 heartbeat checkpoint 会正常上送。

### 1.21 恢复态与后端 current session 对齐第一版

- checkRecoverableSession 现在会额外读取后端 current session。
- 当本地可恢复会话与后端 current session 同时存在时，当前优先采用更新的一份。
- useInterview.sync.test.jsx 已新增恢复态对齐测试，验证会优先选择较新的后端会话。

### 1.22 恢复态冲突处理策略第二版

- checkRecoverableSession 在比较本地会话与后端 current session 时，改为先比较恢复进度，再比较同步时间。
- 当后端只是 heartbeat 更新更晚，但本地 candidateAnswers 更多、currentQuestionIndex 更靠后时，优先保留本地恢复态，避免进度回退。
- useInterview.sync.test.jsx 已新增回归测试，验证不会被“较新但更旧进度”的后端快照覆盖。

### 1.23 页面级恢复 banner 回归补齐

- ChatPage.recovery-banner.test.jsx 现在额外覆盖页面挂载时会触发 checkRecoverableSession。
- 同一文件补齐“恢复”与“放弃”按钮交互，验证会把当前 recoverableSession 正确传给 recoverSession，并调用 dismissRecovery。
- 这一轮先聚焦恢复 banner 页面行为，语音闭环的更完整页面级联动回归继续留在后续切面。

## 2. 当前代码状态判断

基于本轮实际代码，不再建议继续把旧审查报告中的全部问题视为“未处理”。

已经至少部分落地或已明显改善的条目：

- ChatPage 面试状态已引入 useInterview，不再完全是原始的上帝组件形态。
- sendMessage 竞态相关逻辑已通过 useInterview.submitAnswer 串行化处理。
- 可恢复会话已存在。
- 记忆清空已带确认。
- 评分超时与部分 8 维度处理已经改善。

仍需继续推进的重点：

- useVirtualHumanBridge 已承接 SDK 原生 ASR 回调，但语音回答状态仍主要停留在 ChatPage。
- useVirtualHumanBridge 已承接 SDK 原生 ASR 回调，并统一管理语音回答草稿状态。
- 面试会话同步已具备关键 checkpoint + 周期心跳 + 恢复态对齐第一版。

### 1.24 语音回答自动提交 / 手动确认策略开关

- ChatPage 新增 voiceAnswerSubmitMode，支持“手动确认”和“自动提交”两档，并持久化到 localStorage。
- useVirtualHumanBridge 继续负责生成 voiceAnswerDraft，但是否 requiresConfirm 现在由页面当前档位决定。
- ChatPage 在自动提交档下会对无需确认的 voiceAnswerDraft 自动走 submitAnswer；InputBar 也新增了切换按钮。
- 新增 InputBar.voice-answer.test.jsx 与 ChatPage.voice-submit-mode.test.jsx，分别覆盖开关切换与自动提交路径。
- 自动提交触发点已进一步从 ChatPage 监听 effect 下沉回 useVirtualHumanBridge，通过 onAutoSubmitVoiceAnswer 回调降低页面层 glue code。
