一、项目概述
================================================================================

项目名称: LStwinHR - AI智能招聘平台
技术栈:
  - 前端: React 18 + Ant Design 5 + Three.js + Socket.io
  - 后端: Node.js + Express + MySQL
  - AI: DeepSeek API + 本地LLM (Qwen) + 讯飞虚拟人

用户类型:
  - CORP (企业用户): 简历管理、AI面试、候选人筛选
  - PERSONAL (个人用户): 简历管理、模拟面试、能力评估

三、上线前必须解决的问题
================================================================================

--------------------------------------------------------------------------------
3.1 安全性 (优先级: P0 - 阻塞上线)
--------------------------------------------------------------------------------

  [ ] API限流防护
      - 问题: 无请求频率限制,易受DDoS攻击
      - 建议: 添加express-rate-limit中间件
      - 文件: server.js

  [ ] 输入验证中间件
      - 问题: 缺少统一的输入验证,存在注入风险
      - 建议: 使用Joi或Zod进行请求参数验证
      - 文件: 所有routes/*.js

  [ ] CORS白名单验证
      - 问题: 当前允许所有来源访问
      - 建议: 配置生产环境域名白名单
      - 文件: server.js (cors配置)

  [ ] 文件上传安全扫描
      - 问题: 上传文件无病毒/恶意内容检测
      - 建议: 集成ClamAV或云端扫描服务
      - 文件: routes/resume.routes.js

  [ ] Refresh Token机制
      - 问题: 只有Access Token,无刷新机制
      - 建议: 实现双Token认证体系
      - 文件: routes/authRoutes.js

  [ ] 密码重置流程
      - 问题: 用户无法自助重置密码
      - 建议: 添加忘记密码邮件验证流程
      - 文件: routes/authRoutes.js

  [ ] SQL注入审计
      - 问题: 需审查所有SQL拼接语句
      - 建议: 全面使用参数化查询
      - 文件: db.js, routes/*.js

--------------------------------------------------------------------------------
3.2 基础设施 (优先级: P0/P1)
--------------------------------------------------------------------------------

  [ ] 结构化日志系统
      - 问题: 缺少统一的日志记录
      - 建议: 集成Winston或Pino
      - 影响: 无法有效排查生产问题

  [ ] 错误监控
      - 问题: 无异常追踪和告警
      - 建议: 集成Sentry错误监控
      - 影响: 无法及时发现线上问题

  [ ] 健康检查端点优化
      - 问题: 缺少详细的健康状态检查
      - 建议: 添加数据库/Redis/AI服务状态检查
      - 文件: server.js

  [ ] 优雅关闭处理
      - 问题: 无进程关闭时的资源清理
      - 建议: 添加SIGTERM/SIGINT处理
      - 文件: server.js

  [ ] 数据库连接池优化
      - 问题: 连接池配置需针对生产环境调优
      - 建议: 调整connectionLimit, queueLimit等参数
      - 文件: db.js

  [ ] Redis缓存层
      - 问题: 高频数据无缓存,数据库压力大
      - 建议: 缓存用户信息、岗位配置等
      - 新增: redis配置
