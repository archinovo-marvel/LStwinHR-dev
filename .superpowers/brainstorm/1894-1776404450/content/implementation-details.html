视觉平衡 - 实现方案对比

Flex填充方案（选择C）的具体实现方式

方案1：栅格三等分

代码改动

// HomePage.js 第564行
<Col xs={24} sm={12} lg={8} key={index}>
// 原为 lg={6}，改为 lg={8}
// 24列栅格，三等分 = 8列/卡片

优点

  • 简单直接，仅改一个数字
  • 保持现有响应式结构
  • 三等分精确，无空白

缺点

  • 可能显得"太满"
  • 间距固定，无法微调
  • 视觉平衡性较弱
方案2：Flex视觉平衡（推荐）

代码改动

// 1. 替换Row/Col结构
<div className="features-container">
  {features.map((feature, index) => (
    <motion.div className="feature-item">
      <FeatureCard ... />
    </motion.div>
  ))}
</div>

// 2. CSS样式
.features-container {
  display: flex;
  gap: 20px;
  flex-wrap: wrap;
}
.feature-item {
  flex: 1 1 30%; /* 基础30%，可拉伸 */
  min-width: 280px;
  max-width: 400px;
}

优点

  • 视觉平衡，可微调比例
  • 响应式更灵活
  • 间距可控，美观

缺点

  • 改动稍大
  • 需新增CSS样式
  • 需测试响应式

设计建议

基于"视觉平衡"选择，推荐方案2：Flex视觉平衡。实现要点：

  1. 保持现有卡片组件和动画效果
  2. 用flex容器替换Ant Design栅格系统
  3. 设置合适flex比例（如1.2）而非严格三等分
  4. 保留响应式断点（手机端垂直堆叠）
  5. 微调间距和最大宽度，避免"太满"
